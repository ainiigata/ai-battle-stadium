# ゴリすけチャットボット 設計ドキュメント

作成日: 2026-06-13

## 概要

既存の `ai-shindan.html`(生成AI選び診断)を「ゴリスケチャットボット」としてリブランド・UI再設計する。
診断ロジック・データ(CATEGORIES、スコアリング関数、セルフテスト)はそのまま再利用し、画面切り替え型UIを
連続チャット形式のUIに置き換える。マスコットキャラクター「ゴリすけ」(`gorisuke-character.png`)が
吹き出しで質問・結果を語りかける。

## 対象ファイル

- `ai-shindan.html` を直接改修(新規ファイルは作らない)
- `gorisuke-character.png`(1024×1536、ユーザー提供)をリサイズして同階層に保存し、base64で埋め込む
  - 単一HTMLファイルでの完結性(コピー・共有してもそのまま動く)を維持するため
  - `sips` でリサイズ(長辺600px程度)してファイルサイズを抑える

## 1. 会話フロー(連続チャット形式)

```
[ヒーロー画像: ゴリすけ全身]
🦍 やっほー!ゴリすけだよ。AIツール選びを手伝うよ!
🦍 今日は何について知りたい?
   [文章で相談・LLM] [画像生成] [動画生成] [コーディング]
                                  (ユーザー選択 → 右バブル)
🦍 (タイピング...) 主に何に使いたい?
   [選択肢1] [選択肢2] ... 
                                  (ユーザー選択 → 右バブル)
🦍 (タイピング...) 予算は?
   ...(質問を順に繰り返す。LLM/画像=4問、動画/コーディング=5問)
🦍 (タイピング...) よし、診断結果が出たよ!
   [結果カード: 1位/2位/3位・理由・リスク・公式リンク・footerNote]
🦍 [もう一度診断する]
```

- 質問・結果は `state.categoryKey` / `state.questionIndex` / `state.answers` で管理(既存のstateをそのまま使用)
- 「もう一度診断する」を押すとチャット欄をクリアし、ウェルカムメッセージから再開

## 2. メッセージタイプ

| タイプ | 構成 | 用途 |
|---|---|---|
| ボットメッセージ(テキスト) | 円形アバター + 吹き出し(テキスト) | 挨拶・質問・つなぎの一言 |
| ボットメッセージ(クイックリプライ付き) | 上記 + 選択肢ボタン群 | ジャンル選択・各質問 |
| ボットメッセージ(結果カード) | アバター + 吹き出し内に結果カード(既存result-first/result-others相当) | 診断結果表示 |
| ヒーローメッセージ | ゴリすけ全身画像 + テキスト | 会話冒頭の挨拶 |
| ユーザーメッセージ | 右寄せ・ゴールド背景バブル(アバターなし) | ユーザーが選んだ選択肢のラベル表示 |
| タイピングインジケーター | アバター + 「●●●」アニメーション吹き出し(一時的) | ボット発言前の演出(400〜600ms) |

## 3. ビジュアルデザイン

- 配色は既存の `--gold: #fbd784` / `--dark: #0b1d26` テーマを継続(ゴリすけの黄色いフーディと相性が良い)
- フォントも既存(Montserrat / Playfair Display / IBM Plex Sans JP)を継続
- アバター: `gorisuke-character.png` を `background-size`/`background-position` で顔部分にズームし、44px円形に表示
- 吹き出し:
  - ボット: 左寄せ、`var(--dark-card)` 背景 + `var(--border)` 枠、角丸(左上だけ少し小さく「しっぽ」感を出す)
  - ユーザー: 右寄せ、`linear-gradient(135deg, var(--gold), #f5b942)` 背景、`var(--dark)` 文字
- タイピングインジケーター: 3つの小さい円が上下にバウンドするCSSアニメーション
- 新しいメッセージ追加時はチャットコンテナを自動で最下部までスクロール
- 進捗ドット: クイズ中のボット質問メッセージの上に小さく表示(現行の`.progress-dots`をメッセージ内に内包)

## 4. ヘッダー・タイトル

- `<title>`: 「ゴリスケチャットボット | あなたにぴったりの生成AIが見つかる」
- `<h1>`: 「ゴリスケチャットボット」
- サブタイトル: 「ゴリすけと話して、あなたにぴったりの生成AIを見つけよう」

## 5. 技術的な変更範囲

### 再利用(変更なし)
- `CATEGORIES` データオブジェクト(全28サービス)
- `budgetScore` / `extraAdjust` / `scoreService` / `buildReasons` / `rankServices`
- セルフテスト `runSelfTests()`(ロジック不変のため12 pass/0 failを維持)

### 作り直し
- HTML body: ジャンル/クイズ/結果の3画面構成 → 単一の `#chat` コンテナ + メッセージを動的追加
- CSS: `.screen`系を削除し、チャットバブル・アバター・タイピングアニメーション用のスタイルを追加
- エンジンJS:
  - `state` は維持(`categoryKey`, `questionIndex`, `answers`)
  - `showScreen`/`renderGenreScreen`/`renderQuestion`/`renderResults`/`resetToGenre` を、
    チャットメッセージ追加方式の新関数群に置き換え
    - `addBotMessage(contentHtml, { quickReplies, avatar })`
    - `addUserMessage(text)`
    - `showTyping()` / `removeTyping()`
    - `startChat()`(ウェルカム+ジャンル選択)
    - `selectGenre(key, label)`
    - `askNextQuestion()`
    - `selectAnswer(value, label)`
    - `showResultMessage()`(結果カードHTML生成は既存`renderResults`相当のロジックを再利用しつつ、戻り値をHTML文字列として組み立てる形に変更)
    - `restartChat()`
  - `renderResults` は DOM直接操作からHTML文字列を返す関数に変更(チャットバブル内に挿入するため)

### 画像アセット
- `sips -Z 600 gorisuke-character.png` でリサイズ → base64エンコードして `<style>` 内または `<script>` 内の定数として埋め込み

## 6. セルフテストへの影響

- スコアリング・データ・ランキングのロジックは変更しないため、既存の12アサーションは全てそのまま通る想定
- `?test=1` クエリパラメータでの起動は維持

## 7. 手動確認項目

- 4ジャンル全てで会話が最後まで進み、結果カードが表示されること
- 各質問でクイックリプライ選択 → ユーザーバブル表示 → 次の質問へ自動進行
- タイピングインジケーターが表示・消去されること
- 結果カード内の公式リンク(`https://`付き)・理由リスト・リスク表示・footerNoteが正しく表示されること
- 「もう一度診断する」でチャットがリセットされ、ウェルカムメッセージから再開できること
- アバター画像が顔部分にズームされて円形に表示されること
- チャット欄が新規メッセージ追加時に自動スクロールすること
